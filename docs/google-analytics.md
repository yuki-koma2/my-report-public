# Google Analytics 設計・運用ルール

このドキュメントは、`my-report-public` の Google Analytics 設計方針、実装ルール、確認項目を定義します。

## 前提

- Google Analytics は GA4 を前提にする。
- このサイトは GitHub Pages で配信する静的 React SPA であり、サーバーサイドの計測処理は持たない。
- Analytics はサイト改善のための集計用途に限定し、読者個人を特定する用途では使わない。
- Analytics を送信しない環境でも、サイトの表示、ルーティング、レポート閲覧が正常に動く状態を保つ。

## 設定値

- デフォルトの測定 ID は `G-95VGGHF660` とする。
- 別の測定 ID を使う場合は `VITE_GA_MEASUREMENT_ID` で渡す。
- 測定 ID を React コンポーネントや HTML に固定値として直接埋め込まない。
- 測定 ID が空文字の場合は Google Analytics のスクリプトを読み込まない。
- `.env.local` などローカル専用ファイルに測定 ID を置く場合は、Git 管理に含めない。
- 測定 ID 自体は秘密情報ではないが、環境ごとの差し替えを可能にするため環境変数で管理する。

## 実装方針

- Google Analytics の初期化処理は、アプリ本体から分離した小さなモジュールにまとめる。
- 初期化処理は idempotent にし、React の再レンダーや開発時の Strict Mode で重複 script が挿入されないようにする。
- `window.gtag` や `window.dataLayer` を直接触る箇所は、できるだけ Analytics 専用モジュールに閉じ込める。
- Analytics が無効な環境では、計測関数は no-op として動作させ、呼び出し側で条件分岐を増やさない。
- `localhost`、`127.0.0.1`、`::1` では、テストで明示的に許可した場合を除き送信しない。
- gtag の自動 `page_view` は無効化し、SPA 側で手動送信して二重計測を避ける。
- Google Tag Manager や他の広告タグを併用する場合は、このドキュメントに追加方針を追記してから実装する。

## ページビュー計測

- 初回表示だけでなく、SPA のクライアントサイドルート変更もページビューとして送信する。
- 送信する `page_path` は、pathname、search、hash を含める。
- レポート詳細ページでは、URL パスを送信してよいが、本文の全文、検索語句、ユーザー入力値を page view に含めない。
- ルート定義を変更した場合は、ページビュー計測が壊れていないか確認する。

## イベント計測

- イベント計測は、サイト改善に必要な最小限にする。
- イベント名は英小文字、数字、アンダースコアで統一する。
- イベント名は `source_click`、`select_report`、`filter_report_tag` のように、読者の操作を具体的に表す名前にする。
- 送信してよい値は、レポート ID、タグ ID、出典種別など、公開済みコンテンツに由来する識別子に限定する。
- 個人情報、問い合わせ内容、自由入力テキスト、検索語句、IP アドレス、メールアドレス、電話番号、内部メモは送信しない。
- 出典リンククリックは `source_click` として送信する。
- `source_click` では、`report_id`、`source_title`、`source_domain` のような公開コンテンツ由来の値に限定し、リンク先 URL 全体や読者入力値は送信しない。

## プライバシー

- Analytics の設定は、個人を特定しない集計用途に限定する。
- Google シグナル、広告向け機能、リマーケティングなど、読者追跡の性質が強い機能を有効化する場合は、目的、対象地域、同意管理の必要性を事前に確認する。
- Cookie バナーや同意管理が必要になる設定を使う場合は、同意前に Analytics を読み込まない設計にする。
- 同意管理を実装する場合は、同意済み、拒否済み、未選択の状態を分けて扱う。
- 同意を拒否しても、レポート閲覧やサイト内ナビゲーションが制限されないようにする。

## ローカル開発と PR

- ローカル開発では、デフォルト ID があっても localhost 判定により送信しない。
- pull request の preview やローカル検証で本番 GA プロパティへ送信しない。
- 計測の動作確認が必要な場合は、検証用 GA4 プロパティまたは DebugView を使う。
- テスト環境では Analytics を no-op にし、外部ネットワークへ依存しないテストにする。

## テスト・確認項目

Google Analytics の実装や設定を変更した場合は、可能な範囲で次を確認する。

- 測定 ID が空文字の場合に script が挿入されない。
- localhost では明示的に許可しない限り script が挿入されない。
- 測定 ID 設定時に script が一度だけ挿入される。
- 初回表示でページビューが送信される。
- hash route の変更でページビューが送信される。
- 同じ `page_location` のページビューが重複送信されない。
- Analytics 無効時でもレポート一覧、詳細ページ、外部出典リンクが正常に使える。
- イベント計測に個人情報や自由入力値が含まれていない。
- `npm run test` と `npm run build` が成功する。

## 変更時の記録

Google Analytics に関する変更を行う場合は、PR またはコミット本文に次を記録する。

- 何を計測するようにしたか。
- なぜその計測が必要か。
- どの環境変数や GitHub Actions 設定を使うか。
- 個人情報や機密情報を送らないために確認したこと。
- ローカル、テスト、build、本番確認のうち実施した確認内容。

## 実装構成

Analytics 関連の処理は次のように分離する。

```text
src/
└── app/
    ├── analytics.js        # GA 初期化、page view、event 送信の薄い wrapper
    ├── routes.js           # ルート定義
    └── main.jsx            # アプリ起動時の初期化呼び出し
```

実装例を追加する場合も、測定 ID の実値は含めない。
